# Per-Page Visual Inspection Criteria

Use this file during Phase 1B (UX/UI analysis) and Phase 3 (CRO frameworks) to apply page-specific criteria. Generic observations that apply to any e-commerce site are not useful — every finding must be grounded in what's visible in the screenshot AND specific to this store's context.

---

## HOME PAGE — What to Look for

### Above the fold (first viewport — most critical)

**Hero area:**
- [ ] Does the headline answer "what do you sell + for whom" in one read?
- [ ] Is there a clear subheadline or descriptor that adds specificity?
- [ ] Is the primary CTA visible — and is it ONE button, not two competing options?
- [ ] Does the hero image reinforce the value proposition or just look pretty?
- [ ] Is there an urgency or promotion element visible (banner, countdown, offer)?

**Navigation:**
- [ ] Is the logo/brand identity immediately recognizable?
- [ ] Can a new visitor find what they're looking for in the main nav within 3 seconds?
- [ ] Is the search bar visible (critical for catalogs > 50 SKUs)?
- [ ] Is the cart icon visible with item count?

**Visual red flags on HOME:**
- Hero CTA says "Explorar" or "Ver más" → no benefit, no conversion pull
- No price indication anywhere above fold → creates uncertainty for cost-sensitive visitors
- Auto-playing video with sound → immediately alienates users
- Nav with > 7 items → cognitive overload
- Hero image shows people but no product → brand awareness, not conversion
- Three or more CTAs competing in the hero → momentum killer

### Below the fold (rest of the page)

- [ ] Featured categories: visual, labeled, lead to real destinations?
- [ ] Social proof section: review count, ratings, press logos, awards?
- [ ] Bestsellers/new arrivals grid: do the products shown match what GA4 says actually sells?
- [ ] Email capture: is there a compelling offer (not just "subscribe to newsletter")?
- [ ] Trust bar: free shipping, returns, payment methods — is it visible early?

**Site-specific question:** Given this brand's audience and price tier, does the homepage build enough trust for a first-time visitor to click through to a product?

## PDP — FICHA DE PRODUCTO — What to Look for

### Above the fold (first viewport — conversion decision zone)

**Product images:**
- [ ] Is the hero product image large, sharp, and unambiguous?
- [ ] Does it show the product clearly, not just a lifestyle mood shot?
- [ ] Is there a thumbnail strip indicating more images exist?
- [ ] On mobile: is the image swipeable (swiping is the mobile signal for more images)?

**Product name + price block:**
- [ ] Is the product name specific enough? (Not "Blue Shirt" — "Camisa Oxford Slim Fit Azul Marino")
- [ ] Is the price immediately visible without scrolling?
- [ ] If discounted: is the original price crossed out AND the saving amount shown?
- [ ] Is the shipping condition near the price? ("Envío gratis" near the price = conversion boost)

**Variant selector (if applicable):**
- [ ] Can the user select size/color/variant without confusion?
- [ ] Is the selected state visually clear?
- [ ] Are unavailable variants shown as crossed-out (not hidden)?
- [ ] Is a size guide accessible without leaving the page?

**Add to Cart CTA:**
- [ ] Is the ATC button visible above the fold ON MOBILE?
- [ ] Is it the visually dominant element (right color, right size)?
- [ ] Does it say what happens when you click it? ("Agregar al carrito" is better than "Comprar")
- [ ] Is there an express checkout option (Apple Pay, Google Pay, PayPal)?
- [ ] Is there a wishlist/save option as a lower-commitment alternative?

**Visual red flags on PDP:**
- ATC button same color/weight as other page elements → gets lost
- Price only appears after selecting a variant → friction and confusion
- No reviews visible in first viewport → missing trust at decision point
- "Only X left" claim without specific number → fake scarcity, hurts trust
- No return policy/guarantee near the ATC → missing anxiety reducer

### Below the fold

- [ ] Product description: does it lead with benefits, not features?
- [ ] Reviews section: how many? How recent? Are there photos?
- [ ] Cross-sell / related products: are they relevant to this specific product?
- [ ] FAQ or expandable sections: are common objections addressed?
- [ ] Trust badges: SSL, money-back guarantee, certifications?

**Site-specific question:** Given this product's price and the brand's trust level, what is the main anxiety a first-time visitor has at this moment — and does the PDP address it?

## CART — What to Look for

### Immediate view (what user sees right after ATC)

**Cart summary:**
- [ ] Is the product they just added shown clearly — correct image, name, variant, quantity, price?
- [ ] Is the order total visible without scrolling?
- [ ] Is the "Proceed to Checkout" CTA primary, obvious, and above the fold?
- [ ] Is there a free shipping progress bar? ("Add $12 more for free shipping")

**Friction reduction:**
- [ ] Can the user change quantity easily?
- [ ] Can the user remove items easily?
- [ ] Is the promo code field visible but NOT the most prominent thing? (A prominent promo field makes users leave to find codes)
- [ ] Is guest checkout mentioned here? (Many users don't know they can skip account creation)

**Trust signals in cart:**
- [ ] Are payment logos visible? (Reduces payment anxiety at this pre-checkout moment)
- [ ] Is the money-back or return policy mentioned?
- [ ] Is there an SSL or "secure checkout" badge near the total?
- [ ] Is the estimated delivery date shown? (Massive conversion lift)

**Visual red flags in CART:**
- Cart total shows a different price than PDP → immediate distrust
- Checkout CTA says "Continue" instead of "Proceed to Checkout" → ambiguous
- Cart page has full navigation bar → invites users to wander away
- Cross-sell section is larger than the checkout CTA → wrong visual hierarchy
- No trust signals anywhere near the order total → anxiety at peak moment

**Site-specific question:** For this brand's price tier, what's the main reason a user would abandon at cart — price shock, trust, or distraction? Does the cart address that specific risk?

## CHECKOUT — What to Look for

### Step 1 (Information / Shipping)

**Form design:**
- [ ] How many fields are visible on first screen? (>7 without progress indicator = high abandonment risk)
- [ ] Are fields clearly labeled (not just placeholder text that disappears)?
- [ ] Is autofill enabled? (This alone reduces checkout time 30–50%)
- [ ] Is there an address autocomplete / zip code lookup?
- [ ] Is email field first? (Captures lead even if user abandons)
- [ ] Is there a clear step indicator? ("Step 1 of 3: Shipping")

**Account creation friction:**
- [ ] Is guest checkout available? Is it the DEFAULT or buried?
- [ ] If login required: is social login (Google, Apple) offered?
- [ ] Is account creation OPTIONAL and suggested AFTER purchase?

**Trust signals in Step 1:**
- [ ] Is the URL HTTPS visible in the browser? (Users check this)
- [ ] Are SSL/security badges in the checkout header?
- [ ] Is the order summary visible (collapsed if mobile, expanded if desktop)?

### Step 2 (Payment)

**Payment form:**
- [ ] Are all relevant payment methods shown? (Visa, MC, PayPal, Apple Pay, local methods for this market)
- [ ] Is card number field formatted as user types? (1234 5678 9012 3456 format)
- [ ] Is the CVV explained? (Small "?" next to CVV reduces errors)
- [ ] Is there a "Save this card" option for returning users?

**Trust signals in Step 2 (most critical — user is about to enter card data):**
- [ ] Security badge directly above or below the payment form?
- [ ] "Your payment is secured by 256-bit SSL" type message?
- [ ] Money-back guarantee mentioned here?
- [ ] Return policy accessible (opens in modal, NOT new tab)?

**Submit button:**
- [ ] Does the submit button text make the action clear? ("Completar compra" is better than "Continuar")
- [ ] Is the total shown one more time directly above the submit button?
- [ ] Is there a loading state after clicking? (Avoids double clicks)

**Visual red flags in CHECKOUT:**
- Full site navigation still visible → invites users to leave
- Taxes / fees appear for the first time in this step → price surprise = abandonment
- Payment logos shown but Apple Pay / Google Pay NOT actually available → misleading
- Submit button same visual weight as "Back" link → sends users backward accidentally
- No progress indicator → users don't know how many more steps exist

**Site-specific question:** For this market (country, language, audience), what local payment methods are expected — and are they all present? Is this a one-step or multi-step checkout, and is that appropriate for this brand's AOV?
