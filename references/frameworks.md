# CRO Frameworks Reference

## 1. The 4Ps — E-commerce CRO Edition

### PRODUCT

**Core Questions:**
- Is it immediately clear what you're selling and to whom?
- Does the product photography convey quality, scale, and context?
- Is the product name SEO-optimized AND human-readable?
- Are specs, variants, and technical details complete and scannable?
- Is there a comparison table if there are multiple SKUs?
- Are out-of-stock products handled gracefully (notify me, similar items)?

**Photography Checklist:**
- [ ] Minimum 4 images per product
- [ ] White background (detail) + lifestyle/context images
- [ ] Zoom functionality on desktop
- [ ] Swipe gallery on mobile
- [ ] Video or 360° view (conversion +30–40%)
- [ ] Size reference image (person wearing, item next to common object)
- [ ] Images load fast (<1MB webp)

**Description Checklist:**
- [ ] First 2 lines answer "what is it and who is it for"
- [ ] Benefits before features (not specs-first)
- [ ] Scannable: bullets, short paragraphs, bold key words
- [ ] Addresses the main objection or anxiety for this category
- [ ] Includes social proof in copy ("our best-seller", "as seen in X")
- [ ] Includes use cases / occasions

**Variants:**
- [ ] All variants visible upfront (don't hide them behind a click)
- [ ] Unavailable variants shown as struck-through, not hidden
- [ ] Size guide accessible without leaving the page
- [ ] Color/option selection confirms selection visually (selected state)

---

### PRICE

**Core Questions:**
- Is the price immediately visible above the fold?
- If discounted, is savings amount AND % both shown?
- Is there a price anchor (original price struck through)?
- Does the site use psychological pricing (.99, .95)?
- Is the pricing context given (per unit, per kg, per month)?
- Are there trust signals near the price (money-back, guarantee)?

**Price Display Hierarchy:**
1. Discounted price (large, prominent)
2. Original price (struck through, smaller)
3. Savings ("You save €X / XX%")
4. Shipping cost (or "Free shipping" message)
5. Installment option if available ("Only €X/month")

**Pricing Psychology Triggers:**
- **Anchoring**: Show a higher price first, then the actual price
- **Charm pricing**: €19.99 vs €20.00 (perceived value difference)
- **Bundle pricing**: "Buy 2, save 20%" increases AOV
- **Free threshold**: "Add €X more for free shipping" (most powerful AOV lever)
- **Price isolation**: Focus on daily cost ("Less than a coffee per day")
- **Payment plans**: Installments reduce perceived price barrier for high-ticket

**Red Flags:**
- Price only visible after scrolling
- No indication of whether taxes/shipping are included
- Confusing pricing structure (makes user hesitate)
- "Price varies" without explanation

---

### PLACE (Navigation & Findability)

**Site Navigation:**
- [ ] Categories are intuitive and match user mental models
- [ ] Mega menu or clear hierarchy for large catalogs
- [ ] Search box is prominent and functional (especially on mobile)
- [ ] Search returns relevant results + handles typos
- [ ] Breadcrumbs present on all interior pages
- [ ] "Continue shopping" after add-to-cart keeps users in flow

**Product Listing Pages (PLP):**
- [ ] Filters are visible and meaningful (price range, color, size, rating)
- [ ] Sort options: relevance, newest, price low-high, best reviewed
- [ ] Product card shows: image, name, price, rating, key variant selector
- [ ] Add-to-cart from listing (reduces friction for known products)
- [ ] Pagination vs. infinite scroll (test per audience)
- [ ] Items per page: 24–48 is sweet spot

**Mobile Navigation:**
- [ ] Hamburger menu accessible from thumb zone
- [ ] Bottom navigation bar for primary categories
- [ ] Search icon prominent (not buried in menu)
- [ ] Cart icon shows item count
- [ ] No horizontal scrolling

**Internal Linking & Flow:**
- [ ] Related products on PDP (same category)
- [ ] Cross-sell on cart ("Complete the look", "Frequently bought with")
- [ ] Up-sell on PDP (premium version)
- [ ] Post-purchase recommendations on thank you page

---

### PROMOTION

**Promotion Visibility:**
- [ ] Active promotions are in the header or hero (not just a banner deep on the page)
- [ ] Promo codes are easy to find and apply
- [ ] Promotion conditions are clear (minimum spend, exclusions)
- [ ] Free shipping threshold is communicated sitewide
- [ ] Loyalty points / cashback communicated at checkout

**Urgency & Scarcity:**
- [ ] "Only X left in stock" (real data, not fake — user trust is paramount)
- [ ] Countdown timer for limited-time offers (real deadlines only)
- [ ] "X people viewing this right now" (use with caution — can feel fake)
- [ ] "Last purchased X hours ago" (social proof + recency)
- [ ] Flash sale / daily deals clearly marked

**Email & Retention Marketing Signals:**
- [ ] Email capture with value proposition (not just "subscribe to newsletter")
- [ ] Exit-intent popup with compelling offer (10-15% off, free shipping)
- [ ] Post-purchase email series visible in order confirmation
- [ ] Loyalty program mentioned with clear benefit

## 2. AIDA Framework — Per Page Type

### Home Page AIDA

**Attention (above the fold):**
- Hero image/video must convey brand essence in <2 seconds
- Main headline answers: "What do you sell + for whom"
- Subheadline or tagline reinforces value proposition
- Primary CTA is visible without scrolling on mobile
- No auto-playing video with sound
- Logo is recognizable and links to home

**Interest (first scroll):**
- Featured categories are visual, scannable, and complete
- Brand story or USPs communicated briefly (3 icons or 3 sentences max)
- Social proof teaser: review count, awards, press logos
- Bestseller or new arrivals grid
- Each section has a clear purpose (no decorative sections)

**Desire:**
- Editorial content: lifestyle photography, lookbook, use cases
- User-generated content (Instagram feed, real customer photos)
- "As seen in" press logos (Forbes, Vogue, etc.)
- Customer testimonials with name, photo, and rating

**Action:**
- Primary CTA leads to main category or best-selling collection
- Secondary CTA for new arrivals or current promotion
- Urgency element (free shipping bar, countdown, sale announcement)
- Chat or help widget accessible

---

### Product Detail Page (PDP) AIDA

**Attention:**
- Hero product image is large, high-quality, shows product clearly
- Product name is clear, specific (not generic)
- Price is visible immediately
- Add-to-cart button is visible without scrolling (sticky on mobile)

**Interest:**
- First description paragraph: key benefit + who it's for
- Specs and features are tabbed or expandable (not wall of text)
- Size/variant selector is intuitive
- Delivery time and free shipping clearly stated near ATC button

**Desire:**
- Review summary (stars + count) visible near product title
- Featured reviews (with photos if available)
- "X people bought this in the last 24h" (if real)
- Stock scarcity signal ("Only 3 left")
- Return policy visible and reassuring near ATC
- Social proof badges (best seller, award winner, press mention)

**Action:**
- ATC button: large, contrasting color, clear text ("Add to Cart", not "Buy Now" unless 1-click)
- "Buy Now" / Express checkout (PayPal, Apple Pay) as secondary option
- Wishlist / Save for later (low-commitment option)
- Clear what happens after clicking (no doubt about next step)

---

### Cart AIDA

**Attention:**
- Cart summary is clear: items, quantities, prices, total
- Progress indicator (if multi-step checkout)
- Free shipping threshold (progress bar showing how close they are)

**Interest:**
- Product images are shown in cart (reduces doubt about what's being bought)
- Easy quantity change and remove option
- Promo code field (but don't make it the focus — it makes people go look for codes)

**Desire:**
- Trust badges near total: secure checkout, SSL, payment logos
- Money-back guarantee reminder
- Cross-sell ("Customers also bought") — but keep subtle, don't distract from checkout
- Estimated delivery date

**Action:**
- "Proceed to checkout" is primary, prominent CTA
- Guest checkout option visible (friction killer)
- Express checkout buttons (Apple Pay, Google Pay, PayPal) — high conversion
- "Continue shopping" secondary (keep it secondary)

---

### Checkout AIDA

**Attention:**
- Clean layout, minimal distractions (remove header navigation)
- Order summary visible (collapsed on mobile, expandable)
- Trust signals in header (SSL, secure checkout badge)

**Interest:**
- Step indicator shows progress (Step 1 of 3: Shipping)
- Form fields are clearly labeled
- "Why we need this" tooltips for unusual fields
- Autofill and address lookup enabled

**Desire:**
- Payment logos (Visa, MC, PayPal, Apple Pay, local methods)
- Security badge near payment section ("Your payment is secured with 256-bit SSL")
- Money-back guarantee near order total
- Return policy link (opens in modal, not new tab that loses checkout)

**Action:**
- Submit button text is action-oriented: "Place Order" / "Complete Purchase"
- Order total is visible before clicking
- Clear what happens after clicking (email confirmation expected)
- Error messages are helpful (not "invalid field" — say exactly what to fix)

## 3. Cialdini 7 Principles — Scoring Criteria

### Scoring: 0 = absent, 1 = weak, 2 = present, 3 = strong

| Principle | 0 | 1 | 2 | 3 |
|-----------|---|---|---|---|
| Reciprocity | No free value | Generic offer | Free shipping or return | Multiple value adds (gift, guide, bonus) |
| Commitment | No saves | Wishlist only | Account + wishlist | Progress, loyalty, personalization |
| Social Proof | No reviews | <10 reviews | Reviews with ratings | 50+ reviews, UGC photos, verified badges |
| Authority | No credentials | 1 badge | Press + certifications | Industry awards + expert endorsement + press |
| Liking | Corporate/cold | Some brand voice | Human photos + story | Community, values, real team, user stories |
| Scarcity | No signals | Generic "limited" | Real stock counts | Multiple scarcity layers (stock + time + exclusivity) |
| Urgency | No triggers | Always-on banners | Real deadlines | Dynamic countdown + personalized urgency |

## 4. Nielsen's 10 Heuristics — E-commerce Criteria

### H1: Visibility of System Status
- Loading states for cart updates
- "Added to cart" confirmation (toast or modal)
- Checkout step indicator
- Form submission feedback
- Payment processing state

### H2: Match Between System and Real World
- Category names match what users call them
- Product names use common terms, not internal codes
- Pricing in local currency and format (€1.234,99 vs $1,234.99)
- Dates in local format
- "Cart" not "Basket" (or vice versa, per market)

### H3: User Control and Freedom
- Easy cart modification (change qty, remove item)
- "Edit" option at checkout review step
- Back button works without data loss
- "Continue shopping" from cart without losing items
- Easy account edits (address, payment)

### H4: Consistency and Standards
- Same button style for same actions across pages
- Consistent color coding (red = error, green = success)
- Consistent terminology (never mix "cart" and "bag")
- Icons have consistent meaning throughout

### H5: Error Prevention
- Address validation before submission
- Highlight required fields
- Real-time email validation
- Payment field formatting help (card number spacing)
- Warning before leaving checkout with items

### H6: Recognition Rather Than Recall
- Breadcrumbs on all pages
- Recently viewed products
- Saved addresses autofill
- Order history accessible
- Search suggestions based on past behavior

### H7: Flexibility and Efficiency
- Advanced filters for power users
- Quick add-to-cart from listing
- Express checkout options
- Saved payment methods
- Guest checkout (don't force account creation)

### H8: Aesthetic and Minimalist Design
- No competing visual hierarchies
- White space used effectively
- One primary CTA per page section
- Consistent typography (max 2 font families)
- Color palette supports hierarchy (not random)

### H9: Help Users Recognize, Diagnose, and Recover from Errors
- Error messages explain the issue and how to fix it
- Inline validation (not only on submit)
- Out-of-stock message + alternatives shown
- Payment error gives specific guidance
- 404 pages redirect to relevant content

### H10: Help and Documentation
- FAQ accessible from key pages (PDP, cart, checkout)
- Live chat visible during checkout
- Return policy link before purchase
- Order tracking accessible from confirmation
- Contact options clear (not buried in footer)
